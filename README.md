Simple Task System (Web)

Описание проекта



Данный проект представляет собой учебную информационную систему для генерации, проверки и анализа программных заданий с использованием методов машинного обучения. Система предназначена для демонстрации полного цикла работы интеллектуального модуля: от загрузки данных и обучения модели до её практического применения при проверке решений пользователей.



Проект разработан в рамках выпускной квалификационной работы и ориентирован на использование в учебном процессе.



Архитектура проекта



Проект состоит из нескольких логически связанных модулей:



ui/ — графический интерфейс пользователя (панели администратора, преподавателя и студента);



ml/ — модуль машинного обучения и интеллектуальной логики;



data/ — данные приложения (датасеты, база данных);



main.py — точка входа в приложение.



Основная логика машинного обучения вынесена в отдельный модуль, что позволяет разделить прикладную и исследовательскую части проекта.



Модель машинного обучения



В системе используется модель классификации текстовых заданий, предназначенная для автоматического определения типа задания на основе его текстового описания.



Подготовка данных



Для обучения модели используется CSV-датасет со следующей структурой:



task\_text — текст задания;



task\_type — тип задания;



input\_data — входные данные для проверки решения.



Перед обучением выполняется проверка структуры датасета и корректности входных данных.



Используемые алгоритмы



В рамках работы были рассмотрены и обучены несколько моделей машинного обучения:



Логистическая регрессия (Logistic Regression);



Наивный байесовский классификатор (Multinomial Naive Bayes).



Для преобразования текстовых данных в числовой вид применяется метод TF-IDF, позволяющий учитывать важность слов в тексте задания.



Обучение и оценка качества



Обучение моделей выполняется на обучающей выборке с последующей проверкой на тестовой выборке. Для оценки качества используется метрика accuracy.



Для каждой модели рассчитываются значения метрик, которые сохраняются в файл:



ml/models/metrics.json





По результатам сравнения автоматически выбирается модель с наилучшим значением accuracy, которая сохраняется и используется в дальнейшей работе системы.



Сохранение и загрузка модели



Обученная модель сохраняется в файл:



ml/models/model\_task\_classifier.pkl





Загрузка и сохранение модели реализованы через отдельный сервисный модуль, что позволяет повторно использовать модель без повторного обучения.



Использование модели в системе



При проверке решения пользователя система автоматически определяет тип задания с использованием обученной модели машинного обучения. Полученный результат используется для выбора алгоритма проверки решения и отображается пользователю, что позволяет наглядно продемонстрировать применение модели по назначению.



Таким образом, модель машинного обучения является не вспомогательным элементом, а непосредственно участвует в логике работы приложения.



Зависимости



Список необходимых библиотек для работы приложения приведён в файле:



requirements.txt



Запуск проекта



Установить зависимости:



pip install -r requirements.txt





Запустить приложение:



python main.py



Назначение проекта



Проект предназначен для учебных и демонстрационных целей и иллюстрирует практическое применение методов машинного обучения в задачах интеллектуального анализа текстов и автоматизированной проверки программных заданий.
